## 2. What This Repo Does

The toxval R package is designed to extract toxicity threshold estimates—such as No-Effect Concentrations (NECs) or ECx values—from fitted dose–response models. It addresses the challenge of consistently deriving these key endpoints from different modelling frameworks by supporting both frequentist (drc) and Bayesian (bayesnec) model outputs.

Key inputs are fitted dose–response model objects, and the primary outputs are standardised toxicity metrics (e.g. NEC, EC10, EC50) that can be used in environmental risk assessment or guideline derivation.

## 5. Git Workflow

This repo uses a standard open-source fork workflow:

```
upstream (open-AIMS/toxval) → origin (beckyfisher/toxval fork) → local workspace
```

- **`upstream`** (`open-AIMS/toxval`) — the authoritative project repo. Never push directly; contribute only via pull requests from the fork.
- **`origin`** (`beckyfisher/toxval`) — the personal fork on GitHub. Positron's Source Control sync (↑↓) pushes/pulls against this remote.
- **`dev`** is the integration branch. New work branches off `dev`; PRs go from `beckyfisher/toxval` into `open-AIMS/toxval`.

### Keeping `dev` up to date with upstream

```bash
git fetch upstream
git checkout dev
git merge upstream/main
git push origin dev
```

Do this before starting any new feature branch.
